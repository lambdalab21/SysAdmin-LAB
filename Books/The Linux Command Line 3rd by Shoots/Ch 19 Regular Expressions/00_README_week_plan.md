# TLCL Chapter 19 Regular Expressions: Multi-Day Plan

A cleaner phrasing: “Please create Markdown files for the split Chapter 19 plan.”

Recommended schedule:

| Day | File | Focus |
|---|---|---|
| Day 1 | `01_day1_grep_literal_matching.md` | `grep`, literal matching, case sensitivity |
| Day 2 | `02_day2_anchors_dot_brackets.md` | `.`, `^`, `$`, bracket expressions |
| Day 3 | `03_day3_classes_repetition.md` | POSIX classes and repetition |
| Day 4 | `04_day4_extended_regex_grouping.md` | `grep -E`, `+`, `?`, `{}`, `|`, grouping |
| Day 5 | `05_day5_applied_review.md` | mixed review and regex discipline |
| Day 6 | `06_day6_passwd_and_package_output.md` | `/etc/passwd`, `apt list`, package-style output |
| Day 7 | `07_day7_config_files.md` | configuration files: SSH and Nginx examples |
| Day 8 | `08_day8_logs_processes_network_output.md` | logs, process output, network output |

Use Days 1–5 for Chapter 19 itself. Use Days 6–8 to solidify the skill with realistic Linux output.

Core habit:

```text
Predict → run → inspect → explain → revise.
```

A student is ready to move on when he can explain why a regex matched, not just that it matched.
