# TLCL Chapter 25: Starting a Project — Three-Day Reading Guide

A cleaner phrasing: “Create Markdown file(s) for Chapter 25, ‘Starting a Project,’ by Shotts. Depending on how the chapter should be split, make Day 1, Day 2, and Day 3.”

This chapter should be split into **three days**.

| Day | File | Main sections | Main gain |
|---|---|---|---|
| Day 1 | `01_day1_minimal_document_and_first_data.md` | “First Stage: Minimal Document” and “Second Stage: Adding a Little Data” | Build a minimal working script and add one useful report value |
| Day 2 | `02_day2_variables_constants_and_assignment.md` | “Variables and Constants” and “Assigning Values to Variables and Constants” | Understand shell variables, empty variables, quoting, and expansion |
| Day 3 | `03_day3_here_documents_final_lab.md` | “Here Documents” and “Summing Up” | Replace repeated `echo` lines with readable multiline script output |

## How to use these files

Each day has this pattern:

```text
1. Read a section with a specific purpose.
2. Answer concept questions after reading.
3. Predict before typing.
4. Run a small exercise.
5. Inspect the output.
6. Explain what changed.
7. Read the next section only after the exercise has meaning.
```

## Final Chapter 25 standard

He is ready for Chapter 26 only if he can explain:

```text
Why start with a minimal script?
Why print to standard output?
Why redirect output to an HTML file?
What is a shell variable?
What happens when a variable is empty?
Why do quotes matter in assignments?
What is a here document?
When should a here-document delimiter be quoted?
Why must the here-document closing token be exact?
```
